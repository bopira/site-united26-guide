# site-united26-guide

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件的形式存储，无需后端服务即可直接访问。仓库本身不针对任何特定域名、网站或服务，仅作为 HTML 内容的结构化存储和展示空间。

## 目录说明

```
.
├── README.md          # 本说明文件
├── index.html         # 仓库首页入口
├── pages/             # 存放所有 HTML 页面
│   ├── example1.html  # 示例页面
│   └── example2.html
├── assets/            # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── docs/              # 文档或说明性文件（可选）
```

- **pages/**：核心目录，所有归档的 HTML 页面均放置于此。可根据需要创建子文件夹进行分类。
- **assets/**：存放页面引用的静态资源，如样式表、脚本、图片等。建议保持目录结构清晰，便于维护。
- **docs/**：可选，用于存放与页面内容相关的说明、引用或补充资料。

## 页面归档说明

- 所有 HTML 页面均为独立文件，不依赖外部数据库或动态服务。
- 页面之间可通过相对路径互相链接，但建议尽量保持自包含，减少交叉依赖。
- 归档内容以通用、中性为原则，不涉及特定品牌、域名或商业推广信息。
- 页面可包含文本、图片、多媒体等元素，但需确保所有资源均托管于本仓库内或使用公开稳定的 CDN 链接。

## 维护说明

- 如需新增页面，请在 `pages/` 目录下创建新的 HTML 文件，并确保链接和资源引用正确。
- 更新现有页面时，请尽量保留原有结构，避免破坏已有引用关系。
- 若需删除或移动页面，请同步检查并更新相关链接，避免产生死链。
- 鼓励使用有意义的文件名和目录结构，便于后续检索和管理。

## 使用建议

- 本仓库适合用于存放演示页面、技术文档、个人作品集、学习笔记等静态 HTML 内容。
- 可通过 GitHub Pages 或其他静态托管服务直接部署，快速发布为可访问的站点。
- 欢迎通过 Issue 或 Pull Request 提出改进建议。
